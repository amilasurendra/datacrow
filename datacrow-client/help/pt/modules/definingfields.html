Definir campos

Uma parte importante de definir um módulo consiste em definir os campos que lhe pertencem. No ecrã vê duas tabelas, uma com os campos predefinidos (para o tipo de módulo escolhido), outra com os campos recém criados. Os campos predefinidios não podem ser alterados ou removidos. Contudo, pode criar, alterar e remover campos da secção Novo campo (parte inferior do ecrã).



- Remover campo: remove o campo selecionado da secção Novo campo.
- Adicionar campo: abre o diálogo Definir campo para criar um novo campo.
- Alterar campo: pode ser usado para alterar um campo existente. Ao alterar modos existentes, as alterações que pode fazer aqui são limitadas. Em todo o caso, a maioria das alterações pode ser feita usando as definições, como nas Definições gerais do campo.

Criar um campo

Para criar um novo campo clique em Adicionar campo. Abre o seguinte diálogo:

Nome

O nome do campo para ser mostrado no ecrã. Para módulos existentes o nome pode ser substituído definindo um rótulo nas Definições gerais do campo.

Pode ser procurado?

Indica se os utilizadores podem procurar/filtrar por este campo.

Requerido

Pode marcar um campo como requerido. Ao gravar, este campo terá sempre de estar preenchido. O item só será gravado quando todos os campos requeridos estiverem preenchidos. Para módulos existentes altere esta opção nas Definições gerais do campo.

Descritivo?

Quando um campo está marcado como descritivo (ou uma combinação de vários campos), o valor deste campo é usado para representar os itens. Para módulos existentes altere esta opção nas Definições gerais do campo.

É campo chave

A combinação de campos chave será usada para verificar se o item já existe. Leia mais acerca disto. Para módulos existentes altere esta opção nas Definições gerais do campo.

Separador do formulário de item

Define em qual separador do formulário de item será colocado o campo. Para módulos existentes altere esta opção nas Definições gerais do campo.

Comprimento máximo do texto

O comprimento máximo dos dados para este campo (em carateres).

Tipo de campo

Há uma grande variedade disponível de tipos de campos e muitos falam por si próprios. Contudo, os campos de referência, única ou múltipla merecem mais alguma explicação. Quando seleciona um destes campos, vai ficar ativa uma lista de referências. Com este tipo de campos, cria uma referência a outro módulo. Se selecionar um campo de referência única, o Data Crow cria uma lista de itens do módulo selecionado (por exemplo, as categorias de programas). Só um destes itens pode ser selecionado. Se optar por um campo de referência múltipla, o campo pode conter mais de um item de propriedade (por exemplo, os géneros musicais para CDs de áudio). É criado um campo com um diálogo de seleção de item.

Outra nota sobre campos de referência: quando faz referência a um módulo de propriedade, os itens deste módulo podem ser geridos no menu Ferramentas. Uma referência a qualquer outro tipo de módulo torna o módulo referenciado um submódulo do módulo atual. Este é o caso, por exemplo, dos atores no módulo Filmes. O módulo Atores é referenciado pelo módulo Filmes, tornado-se um submódulo de Filmes. O submódulo é mostrado na barra de módulos quando seleciona o módulo que o referencia.
