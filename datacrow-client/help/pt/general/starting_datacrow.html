Iniciar a aplicação

Obviamente que conseguiu iniciar o Data Crow, senão não estaria aqui, a ler o ficheiro de ajuda. Contudo, há algumas opções de arranque que podem ser benéficas para si.

Memória disponível

Ou iniciou o Data Crow usando o ficheiro "datacrow.exe" ou está a usar o comando Java (direta ou indiretamente) "Java -jar datacrow.jar". Primeiro que tudo, o Java (não o Data Crow) vai sempre limitar a quantidade de memória disponível para o. Se receber erros OutOfMemoryErrors, avise-me, mas tente também atribuir mais memória ao Data Crow desta forma:
exemplo: Java -Xmx512m -jar datacrow.jar

Usando múltiplas instalações/ambientes

Se usa múltiplas instalações do Data Crow tem várias escolhas a fazer:
- Defina uma pasta de utilizador numa partilha de rede. Cada uma das instalações do Data Crow vai apontar para esta pasta. Isto é estabelecido selecionando a mesma pasta de utilizador em todas as instalações do Data Crow.
- No caso de correr várias instâncias do Data Crow numa só máquina, pode usar a mesma pasta de utilizador (partilhando as mesmas definições e dados) ou atribuir uma pasta de utilizador diferente a cada instalação (parâmetro -userdir).
- Além do acima descrito, pode partilhar a pasta de utilizador entre as várias instalações mas forçá-las a usar uma base de dados diferente por instalação (parâmentro -db).
Os parâmetros requeridos são explicados no próximo parágrafo.

Parâmetros de arranque

Cada um dos parâmetros listados abaixo pode ser fornecido ao iniciar o ficheiro datacrow.exe file ou ao usar o comando Java para iniciar o Data Crow.

Usar uma base de dados diferente

Por predefinição, o Data Crow usa uma base de dados chamada "dc".
Para usar uma base de dados diferente (para testes, por exemplo) pode usar o parâmetro seguinte: -db:<database name>.
Exemplo: Java -Xmx512m -jar datacrow.jar -db:dc_new

Especificar a pasta de utilizador

Use este parâmetro para especificar uma determinada pasta de utilizador. Pode iniciar múltiplas instâncias do Data Crow usando este parâmetro. Adicionalmente, o caminho de utilizador selecionado não será armazenado no ficheiro datacrow.properties. Parâmetro: -userdir:<pasta utilizador>
Exemplo: Java -Xmx512m -jar datacrow.jar -userdir:C:\Data Crow\Dados
Também pode tornar o caminho relativo à pasta de instalação do Data Crow fornecendo o caminho como './nome da pasta'.
Exemplo: -userdir:./dados

Especificar a pasta de instalação

No caso de o Data Crow não conseguir encontrar a pasta de instalação, pode especificá-la manualmente.
Pode acontecer noutras plataformas que não Windows. Parâmetro: -dir:<caminho da instalação>
Exemplo: Java -Xmx512m -jar datacrow.jar -dir:/utilizador/inicial/datacrow

Iniciar o servidor Web

Pode iniciar o servidor Web sem abrir a aplicação principal usando o parâmetro -webserver. Este parâmetro pode ser combinado com o parâmetro de autorizações para contornar os ecrãs iniciais (como o ecrã inicial e o diálogo de início de sessão): -webserver -credentials
Exemplo: Java -Xmx512m -jar datacrow.jar -webserver -credentials:eu/12345

Especificar as autorizações de início de sessão

Como indicado acima, esta opção pode ser usada para contornar o diálogo de início de sessão. Só é realmente lógico utilizá-lo combinado com o comando webserver. Parâmetro: -credentials:utilizador/palavrapasse
Exemplo: Java -Xmx512m -jar datacrow.jar -credentials:eu/12345

Informação adicional para a sessão (debug)

Anexar -debug ao comando de arranque do Data Crow para registar informação de erros. Parâmetro: -debug
Exemplo: Java -Xmx512m -jar datacrow.jar -debug

Ocultar o ecrã inicial no arranque

Para ocultar o ecrã inicial no arranque do Data Crow pode fornecer o parâmetro nosplash. Parâmetro: -nosplash
Exemplo: Java -Xmx512m -jar datacrow.jar -nosplash

Limpar todas as definições

Para repor o Data Crow nas definições originais forneça o seguinte parâmetro: -clearsettings
Exemplo: Java -Xmx512m -jar datacrow.jar -clearsettings
